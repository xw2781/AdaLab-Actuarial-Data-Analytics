# Release 0.2.4

Released on 2026-05-11.

## User-Facing Changes

### Features
- **project-settings**: Add a selected-level component tray to Reserving Class Types formulas.
  - Users can drag or click components into the formula field as double-quoted tokens with plus operators, see components already present in the formula highlighted, and right-click the tray to replace the formula with all selected-level components.
  - The component tray includes selectable + and - operator chips, styled like review-mode operators, before the component values; one operator is active at a time and remains active for subsequent component adds until another operator is selected or the editor closes.
  - Formula review and component tray operators use light green for + and light yellow for -.
  - Formula editing now separates plain-text edit mode from review mode, where components and operators render as rounded tokens and component drops are accepted.
  - Review mode renders operators as larger SVG controls, displays multiplication as X, lets the Reserving Class Type editor be resized, and keeps the component tray inside an auto-sizing Formula frame with a frameless Show all components / Hide all components toggle that includes a rotating arrow SVG.
  - Formula review and Apply normalize repeated and trailing operators before rendering or validation.
  - Dataset Types and Reserving Class Types floating editor close buttons use the same SVG close icon as the main app titlebar.
  - Formula components already present in review mode are hidden from the tray, and dragging a review-mode component pill back to the tray removes it from the formula while normalizing adjacent operators.
  - Formula review renders known values as neutral no-fill pills sized consistently with the component tray, keeps operator circles the same height as component pills, shows calculated values with the reserving-class filter picker's stacked SVG marker, renders source-derived values without markers, and marks unknown values with a light red fill.
  - Components whose names contain slash or dash characters can be clicked or dragged as quoted tokens using an in-page drag payload without being removed by formula cleanup, and component drags are blocked from dropping into editor fields other than Formula.
  - The component tray right-click menu can replace the formula with all selected-level components or only source-derived imported items.
  - The EEX Formula field now has the same selected-level component tray and tokenized review/edit rendering as Formula, defaulting the tray to hidden each time the reserving class type editor opens and targeting EEX Formula for click, drag, drag-back removal, and right-click select-all actions.
  - Dragging a rendered formula component pill outside its Formula or EEX Formula box removes that component and normalizes adjacent operators.
  - Single-clicking a rendered formula component pill in review mode removes it with a short ghost-pill trajectory animation toward the matching component tray, while double-clicking the rendered formula opens raw text editing.
  - Single-clicking a component tray item to add it now shows a matching ghost-pill trajectory animation toward the target formula review box.
  - Project Settings right-click menus now share a single context-menu font-size control for consistent menu text across project, folder, row, and formula component menus.
  - The formula field auto-sizes as components are added and Escape now leaves formula editing without closing the reserving class type editor.
- **dfm**: Add a DFM RPC Bridge sync workflow with local/remote timestamp comparison.
  - DFM tabs now include a Sync button in the path bar that sends Details-page context through the RPC bridge and waits for a returned DFM JSON.
  - DFM now shows a Dataset-style `Loading DFM Settings` spinner popup while saved local method settings are being read and applied.
  - The taller draggable and resizable `Compare DFM Versions` window shows local and remote `last modified` JSON times without JSON file paths, with Local fixed as the framed left card title and Remote Server fixed as the framed right card title, primary actions labeled `Keep Using Local` or `Use Remote Version`, resizing comparison cards with Notes previews that expand up to 300px and scroll without truncating long notes, plus snapshots of ratio selections, average formulas, and full notes text, with a `NEW` seal on the newer card, wider ratio-pattern cells colored dark grey for common exclusions, green for newly excluded cells, and red for removed exclusions with card-specific legends, Ratios-table origin/development hover tooltips, and red/green note text differences.
  - After the user selects a version action, the large comparison window closes and a smaller same-style DFM Sync message box shows concise waiting/final result messages, with `Esc` and outside-click close support when the message box is not busy.
  - `Keep Using Local` now keeps the local DFM JSON unchanged and removes the remote RPC JSON without sending a `Function = SyncDFM` request.
  - DFM method JSON now uses canonical keys only, including `ratio pattern` for ratio selections, `origin labels` and `development labels` for ratio tooltips, `origin length` and `development length` for stored period lengths, `ultimate vector` for the stored Results vector, and `last modified` for GUI save time.
  - DFM can now restore RPC-returned average selections when the remote JSON sends only hard-coded `User Entry` summary rows by inferring standard Volume/Simple summary row definitions from `average formulas` labels and skipping unresolved labels.
  - Desktop JSON saving now preserves canonical DFM keys such as `ratio pattern`, `ultimate vector`, and `last modified`.
  - DFM method JSON no longer writes `summary hidden`; the average formula table now keeps at least one row when deleting rows.
  - Local DFM method filenames now use `DFM@<ReservingClass>@<Name>.json`, making Name the sole DFM instance identity within a project/reserving-class pair; RPC bridge response/status filenames remain length-qualified.
- **shell**: Show the Windows username in the home sidebar brand with a generated initial icon.
  - Added a docs preview page for every supported home brand initial color.

### Improvements
- **build**: Installer progress now advances while ArcRho files are installed instead of staying at 10% through most of setup.
  - The Windows installer now uses NSIS's built-in file installation path for the single-architecture build so progress and file details update during the main install phase without opening a helper console window.
  - The install-page status text now follows the actual progress bar percentage while ArcRho files are being installed.
  - The installer details panel now stays enabled during file installation so users can see ongoing file-install activity instead of a static preparation message.
- **shell**: Floating window minimize now docks the window without stealing focus.
  - Clicking the floating window minimize button now docks that tab at the end of the main tab strip instead of collapsing it in place.
  - The docked tab is not activated by the minimize action; if another floating window remains, the top layered floating window becomes active, otherwise the prior docked tab stays active.
- **notes**: Change Notes file links to use a right-click menu.
  - Detected file paths in Dataset and DFM Notes now show `Open File` and `Copy File Path` from a right-click menu instead of opening on left click.

### Fixes
- **shell**: Make Project Settings reloads pick up refreshed frontend files more reliably.
  - Clear Cache & Reload now reloads the shell with a fresh timestamped UI URL after clearing Electron cache/storage.
  - Project Settings iframes now include the shell UI version parameter, matching other feature pages during cache-clearing and reload flows.
- **dfm**: Fix DFM version compare ratio snapshots for large triangles.
  - The DFM RPC Bridge comparison now sends the full ratio pattern and labels to the frontend instead of truncating the snapshot.
  - Ratio snapshot cells compact for large matrices so 40x40 patterns remain visible in the local/remote comparison cards.
  - Cells with `2` status inside the inferred active triangle diagonal are shown with a light yellow missing-data fill instead of being hidden as outside-triangle cells.
- **dfm**: Stop DFM settings loading animation on tab activation.
  - Switching back to an existing DFM tab now refreshes edit-state reporting without triggering a local method lookup or the Loading DFM Settings popup.
- **dfm**: Remove the DFM saved-name index file and dropdown lookup behavior.
  - DFM no longer creates or reads `dfm_method_names.json` or the browser-local cached DFM Name key; method lookup now uses the current Details fields and the standard local DFM JSON path only.
  - The Details Name field is now a plain text input without the saved-name dropdown/delete controls, avoiding conflicts where indexed lengths could override manual Origin or Development Length changes.
