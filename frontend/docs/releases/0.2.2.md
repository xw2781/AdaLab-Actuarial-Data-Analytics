# Release 0.2.2

Released on 2026-05-06.

## User-Facing Changes

### Features
- **settings**: Moved workspace path settings into the user's AppData profile with a simpler workspace root and subpath schema.
  - The app now uses %APPDATA%\ArcRho\workspace_paths.json and /workspace_paths without carrying legacy ui_config aliases.
  - Local app prefs and app-server cache now use %APPDATA%\ArcRho subfolders without a WebUI nesting folder.
  - First-run installs use built-in defaults until Server Connection is saved.
  - The Server Connection window can browse for the root path and automatically checks drives D: through Z: for an ArcRho Server folder during first-time setup.
- **shell**: Redesign the Home page with a sidebar and icon-based launch cards.
  - Adds SVG icons and a responsive grid layout for the main ArcRho components.
  - Adds a Settings menu refresh command for reloading the active page with a cache-busted document request.
- **shell**: Added in-shell floating windows for main tabs dragged down from the tab strip.
  - Floating tabs keep their live iframe session, are removed from the main tab strip while floated, and can be moved, resized, minimized, closed, or docked back with the title-bar square/max icon or a title-bar double-click.
  - Floating window drag and resize now keep pointer control when the cursor crosses iframe content.
  - Floating and docking now preserve iframe DOM ownership so pages do not refresh or lose in-progress input values during layout changes.
  - Floating windows use rounded corners on Windows 11 to match the main app frame styling.
  - Overlapped floating windows now layer iframe bodies with their shell chrome so background windows do not show through foreground window content.
  - Clicking inside a floating or docked iframe page activates that tab, so overlapped background windows can be brought forward from their visible content.
  - The first click inside an inactive iframe only activates the tab/window and is suppressed before page controls receive it.
  - The iframe host now passes through empty areas so Home remains interactive behind floating windows.

### Improvements
- **shell**: Removed detached window actions for shell tabs and DFM Results.
  - Tabs now remain inside the main ArcRho shell; the previous standalone detached-window entry points and menu actions were removed.
- **shell**: Main tab dragging now switches between tab reordering and floating-window preview based on vertical distance from the tab strip.
  - Dragging a tab about 30px below the strip shows the floating-window preview, while dragging back near the strip hides the preview and resumes tab arrangement before release.
  - The floating window top-right resize hitbox stays clear of the title-bar window controls.
  - Floating windows can now be resized from each edge as well as each corner.
  - Floating windows can be moved partially beyond the left or right app-shell edge while keeping a reachable portion inside the app.
  - Floating and docking now use a short scale/fade animation while preserving the existing iframe session.
  - Dragging a floating window to the left or right shell edge now shows a half-frame snap preview and releasing there snaps the window into that half.
  - After a half-frame snap, dragging the snapped title bar beyond a small threshold restores the previous floating size before moving.
  - New floating windows created by tab pop-out use a 16:10 default size.
  - After the user manually resizes any floating window, new floating windows in the same app session reuse that last adjusted size.
  - The dragged tab now disappears from the tab strip while pop-out preview mode is active, avoiding clipped tab remnants during the drag.
  - Dragging a floating window title bar near the top tab strip now highlights a dock target, shows a tab-order placeholder, and releasing there docks it back into the previewed order.

### Fixes
- **dfm**: DFM method JSON saves no longer repeat the project name in filenames because the project folder already identifies the project.
  - DFM UI loading now checks the new standard method JSON path only, while the name index is used for saved-name metadata such as origin and development lengths.
  - DFM CSV export remains project-folder scoped and keeps its ArcRhoTri-compatible filename format.
- **project-settings**: Project Settings no longer uses legacy folder, Preload, or project settings path columns from the project map.
  - Project map saves now keep project name and table path data while dropping obsolete Folder, Preload, Project Settings, and Settings Profile columns.

## Internal Notes

- **runtime**: Renamed runtime APIs, schemas, cache IDs, and protocol names to ArcRho.
  - Renamed runtime endpoints to /arcrho paths and renamed request models to ArcRho*.
  - Renamed the backend runtime service to arcrho_runtime_service.py.
  - Updated external request protocol names, cache prefixes, and generated dataset IDs to ArcRho names.
- **shell**: Split the shell entrypoint into focused shell modules.
  - Reduced ui_shell.js to bootstrap and controller composition.
  - Moved shell state, tab actions, tab strip rendering, content hosting, menus, hotkeys, preferences, lifecycle, and host controls into separate modules.
